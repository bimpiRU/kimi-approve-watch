# Jarvis × KAW × Kimi — план объединения (тестовая ветка)

Цель: единый управляющий терминал, где **главагент** (интерактивная сессия Kimi CLI)
ставит задачи агентам-исполнителям, KAW обеспечивает автоапрувы и стабильность системы,
Jarvis даёт голосовой/графический фронтенд.

## Что уже есть

- **kimi-approve-watch (KAW)** — автоапрув диалогов Kimi, стабилизатор ПК, автозапуск, логи.
- **jarvis-assistant** — голосовой помощник (Python/Tkinter, Vosk офлайн), уже содержит
  `kimi_integration.py` (открывает вкладку WT с kimi) и `kaw_integration.py` (дёргает скрипты KAW).
- **Эта ветка** — `hub/hub.ps1`: прототип хаба главагента.

## Прототип в этой ветке

```
hub/agents.psd1   — реестр агентов (роль, рабочая папка, режим разрешений)
hub/hub.ps1       — status / agents / dispatch <agent> "задача" / runs / result / kaw
hub/runs/         — логи запусков агентов
```

Главагент вызывает `hub.ps1 dispatch sysadmin "проверь драйверы"` — агент стартует
как `kimi -p` с ролью из реестра в своём WorkDir, вывод пишется в `hub/runs/`.
KAW продолжает апрувить диалоги в интерактивных окнах; у dispatch-агентов режим `--auto`.

## Варианты полного объединения

### Вариант 1. Хаб внутри KAW (реализован здесь)
- PowerShell-ядро: `hub.ps1` + реестр агентов. Jarvis вызывает hub как подпроцесс.
- Плюсы: минимум кода, уже работает, KAW и хаб в одном репо/автозапуске.
- Минусы: нет голоса/UI из коробки, оркестрация «по файлам».

### Вариант 2. Ядро в Jarvis (Python)
- Диспетчер агентов переносится в jarvis-assistant (там уже есть kimi/kaw-интеграции):
  `hub.ps1 dispatch` → Python-модуль `jarvis/hub.py` с той же семантикой.
- Плюсы: голосовое управление агентами («Джарвис, пусть кодер починит тесты»), Tkinter UI.
- Минусы: зависимость от Python-окружения, KAW остаётся отдельным сервисом.

### Вариант 3. Полный мердж в новый репозиторий `jarvis-hub`
- Общая шина задач (файловая очередь JSON: `queue/`, `runs/`, `state/`), три модуля:
  - **kaw-service** — апрувы + стабилизатор (как сейчас, без изменений);
  - **agent-runtime** — запуск/надзор за kimi-агентами, роли, таймауты, ретраи;
  - **jarvis-front** — голос + UI, команды в шину.
- Плюсы: чистая архитектура, независимые модули, расширяемо (web-UI через `kimi server`).
- Минусы: самый долгий путь; имеет смысл после обкатки варианта 1.

## Рекомендация

Обкатать вариант 1 (эта ветка) → голосовые команды через существующий Jarvis (вариант 2
точечно: одна команда «поручи агенту») → приживётся — выносить в `jarvis-hub` (вариант 3).
