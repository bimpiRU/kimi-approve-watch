# Changelog

## [0.3.0] — 2026-07-18

### Добавлено
- `install-service.ps1` — установка как фонового "сервиса" Windows с gate-подтверждением: сервис стартует при включении ПК, но watcher/stabilizer запускаются только после вашего «Да» при входе в Windows.
- `service-runner.ps1` — фоновый раннер, ожидающий сигнала подтверждения (`.service-go`).
- Watcher: `AutoApproveSelf` — явный автоапрув в собственном окне watcher-а (главный терминал).
- Watcher: `FastMode` — уменьшенные задержки, более быстрая реакция на диалоги.
- Watcher: `ApproveKey = 'auto'` — автоопределение номера варианта «Approve once» из текста диалога.
- Watcher: профиль `generic` для любых агентов с похожими диалогами.
- Stabilizer: `AutoCleanTemp` — автоочистка `%TEMP%` при нехватке места на диске.
- Stabilizer: `QuietHours` — тихие часы без уведомлений (например, `"23-07"`).
- Stabilizer: кроссплатформенные уведомления (Windows tray, macOS `osascript`, Linux `notify-send`).
- Stabilizer: расширенный набор productivity tips по формулировке промптов.

### Улучшено
- Watcher: уменьшены задержки фокуса и после нажатия; диалоги распознаются быстрее.
- Watcher: более гибкие профили агентов (маркер `kimi` вместо строгого `kimi-for-coding`).
- Stabilizer: приоритеты неактивных агентов работают мягче и логируют переходы.
- `kaw.ps1` и установщики обновлены до версии 0.3.0.

## [0.2.4] — 2026-07-18

### Улучшено
- Watcher: интервал сканирования по умолчанию снижен с 10 до 5 сек, задержки при апруве уменьшены (быстрее реакция)
- Stabilizer: всплывающие советы по промтам (`PromptTips`) каждые N минут через системный tray-баллон
- Stabilizer: управление приоритетами неактивных агентов (`ManageAgentPriority`) продолжает работать

## [0.2.3] — 2026-07-18

### Добавлено
- `ManageAgentPriority` в стабилизаторе: процессы `kimi`, неактивные более 2 часов, автоматически понижаются до `BelowNormal` и возвращаются к `Normal` при активности
- История CPU агентов хранится в `agent-cpu.history.json`

### Изменено
- `start-all.ps1` передаёт `ManageAgentPriority` из конфига

## [0.2.2] — 2026-07-17

### Исправлено
- `watch-approve-launcher.ps1`: устойчивый парсинг аргументов через `Start-Process -Wait` — убраны ошибки splatting/switch при автозапуске
- Корректная передача повторяющихся `-ExcludeHwnd` из конфига

### Добавлено
- `NoSelfSkip` в наблюдателе: апрувить и в окне, где обсуждается сам watcher (полезно при управлении из той же сессии)
- Подробнее описано исключение личных вкладок по hwnd в README (RU/EN)

## [0.2.0] — 2026-07-16

### Добавлено
- `kaw.ps1` — единая команда управления: start/stop/restart/status/log/enable/disable/config/windows/version
- Выбор варианта апрува: `-ApproveKey 1|2|3` (по умолчанию `1` — одноразовый)
- Конфиг `kaw.config.psd1` (+ `kaw.config.example.psd1`): все параметры обоих модулей, работает и при автозапуске; CLI важнее конфига
- `quickstart.sh` — установка одной командой из Git Bash / MSYS2 / WSL
- Английская версия README (`README.en.md`)
- Профиль агента `claude` (экспериментально): `-Agents 'kimi,claude'`

### Улучшено
- Мягкость: опция возврата фокуса прежнему окну (`-FocusRestore`, по умолчанию выключено)
- Лёгкость: чтение только `TermControl`-элементов UI Automation вместо всего дерева окна
- Оба модуля работают с приоритетом BelowNormal
- `uninstall.ps1`: ожидание завершения процессов (до 40 сек) вместо фиксированной паузы
- Лаунчер: завершается с записью в лог, если целевой скрипт отсутствует
- `start-all.ps1` собирает аргументы модулей из конфига
