2. Поручить задачу агенту

Выбери агента → впиши задачу → «Отправить». Агент стартует фоном (kimi -p с его ролью в его папке). Один активный запуск на агента: повторная отправка ответит «агент занят». Результат — клик по запуску в таблице. Зависшие: кнопка убить зависшие (prune).

Всё хранится в hub/node/data/settings.json, ключи OAuth — в hub/node/.env (в git не попадают).

4. Вход по GitHub / Google

  1. GitHub: Settings → Developer settings → OAuth Apps → New. Homepage URL: http://127.0.0.1:8787, callback: http://127.0.0.1:8787/auth/github/callback.
  2. Google: console.cloud.google.com → Credentials → OAuth client ID (Web). Redirect URI: http://127.0.0.1:8787/auth/google/callback.
  3. Вставь Client ID/Secret на вкладке «Авторизация» + AUTH_ALLOW (твой логин GitHub и/или Google-почта через запятую).
  4. Как только задан хотя бы один провайдер — пульт начинает требовать вход. Без ключей работает открытый локальный режим.

5. Telegram-управление (с телефона)

  1. В @BotFather: /newbot → получи токен.
  2. Вставь токен в Настройки → Авторизация (TELEGRAM_BOT_TOKEN), нажми «↻ Перезапуск».
  3. Напиши боту /start — в консоли сервера появится твой chat id; впиши его в TELEGRAM_CHAT_ID и снова перезапусти. Теперь бот отвечает только тебе.

Команды бота: /status, /runs, /dispatch <агент> <задача>, /prune, /restart, /help. Бот сам присылает уведомления о завершении задач агентов (✅/❌ с хвостом лога) и о старте сервера.

6. Перенос на сервер

HOST=0.0.0.0 PORT=8787 node server.js        # слушать снаружи
# или Docker:
docker build -t jarvis-hub hub/node
docker run -p 8787:8787 --env-file hub/node/.env jarvis-hub

При выносе наружу: обязательно включи OAuth, поменяй callback-URL на домен сервера, поставь перед пультом reverse proxy с HTTPS (Caddy/nginx). Агенты и KAW работают только на Windows-машине — на Linux-сервере пульт поднимется, но панели KAW/G-Helper будут пустыми (это норма: серверная роль — только веб+GitHub, либо держи его на этом ПК).
