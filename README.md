# Kimi Approve Watch

![PowerShell](https://img.shields.io/badge/PowerShell-5.1-5391FE?logo=powershell&logoColor=white)
![Windows](https://img.shields.io/badge/Windows-10%2F11-0078D6?logo=windows&logoColor=white)
![License](https://img.shields.io/badge/license-MIT-green)

> Автономный наблюдатель, который сам жмёт **«1 → Approve once»** в диалогах подтверждения [Kimi CLI](https://github.com/MoonshotAI/kimi-cli), пока вы заняты своими делами.

**EN:** A watchdog that auto-approves the `Approve once` permission dialogs of Kimi CLI running in Windows Terminal, keeps the PC awake, and asks for your confirmation at logon before starting.

---

## Зачем

Kimi CLI при опасных действиях (запуск команд, правка файлов) показывает интерактивный диалог:

```
Run this command?
  1. Approve once
  2. Approve always
  3. Reject
  1/2/3/4 choose
```

Если агентов запущено несколько, диалоги приходится подтверждать постоянно. Наблюдатель сканирует все окна Windows Terminal каждые 10 секунд, находит активный диалог и нажимает `1` за вас. Выбирается только **Approve once** — никаких «approve always».

## Возможности

- **Автоапрув** диалогов `Approve once` во всех окнах Kimi CLI (каждые 10 сек)
- **Шлюз подтверждения при входе в Windows** — работа начинается только после вашего «Да»
- **Keep-awake** — не даёт ПК заснуть и погасить дисплей, пока наблюдатель работает
- **Антидубль** — мьютекс не позволяет запустить два наблюдателя одновременно
- **Чистка мусора** — убирает символ `1`, который TUI иногда оставляет в строке ввода
- **Точное срабатывание** — смотрит только хвост буфера терминала, свёрнутые окна не трогает
- **Самовосстановление** — лаунчер перезапускает наблюдателя при падении
- Установка и удаление в один скрипт, без сторонних зависимостей

## Требования

- Windows 10/11
- Windows Terminal (Kimi CLI должен работать именно в нём)
- PowerShell 5.1 (встроен в Windows, ничего ставить не нужно)

## Быстрый старт

```powershell
git clone https://github.com/bimpiRU/kimi-approve-watch.git
cd kimi-approve-watch
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

Установщик предложит три режима автозапуска:

| Режим | Что делает | Права админа |
|---|---|---|
| `gate` | При входе в Windows показывает окно подтверждения — наблюдатель стартует только после «Да» | Да (UAC при установке) |
| `startup` | Ярлык в «Автозагрузке», стартует сразу и молча | Нет |
| `none` | Без автозапуска, просто запустить сейчас | Нет |

## Управление

```powershell
.\status.ps1         # жив ли наблюдатель + хвост лога
.\stop-watcher.ps1   # остановить (через файл STOP, мягко)
.\uninstall.ps1      # остановить и снять автозапуск
.\show-windows.ps1   # список окон терминала: hwnd, заголовок, наличие диалога
```

## Настройка

`watch-approve.ps1` можно запускать и напрямую с параметрами:

```powershell
.\watch-approve.ps1 -IntervalSeconds 5          # сканировать каждые 5 сек
.\watch-approve.ps1 -ExcludeHwnd 3344318        # не трогать окно с этим hwnd (своё!)
.\watch-approve.ps1 -NoKeepAwake                # не блокировать сон
.\watch-approve.ps1 -Once                       # один цикл и выход (для теста)
```

hwnd своего окна подскажет `show-windows.ps1` — исключите его, если не хотите, чтобы наблюдатель подтверждал диалоги и в вашей личной сессии.

## Как это работает

1. Раз в `IntervalSeconds` секунд перечисляются окна класса `CASCADIA_HOSTING_WINDOW_CLASS` (Windows Terminal).
2. Через UI Automation читается текстовый буфер каждого окна (`TextPattern`).
3. Если в последних 15 строках есть и `Approve once`, и `1/2/3/4 choose` — это активный диалог.
4. Окно выводится на передний план (`AttachThreadInput` + `SetForegroundWindow`), отправляется клавиша `1` (`SendKeys`).
5. Если после апрува в строке ввода остался мусорный символ `1` — отправляется Backspace.
6. Свёрнутые окна, окна из `-ExcludeHwnd` и сессии, где обсуждается сам этот бот, пропускаются.

## Почему не служба Windows?

Службы работают в сессии 0 и **не видят окна пользователя** — UI Automation и SendKeys там бесполезны. Поэтому наблюдатель запускается в интерактивной сессии: либо через задачу планировщика с триггером «при входе в систему» (режим `gate`, с окном подтверждения), либо через «Автозагрузку` (режим `startup`). Это и безопаснее: без вашего входа и подтверждения автоапрув не работает.

## Файлы

| Файл | Назначение |
|---|---|
| `watch-approve.ps1` | Ядро: сканирование окон, апрув, keep-awake |
| `watch-approve-launcher.ps1` | Обёртка: перезапуск при падении, антидубль |
| `watcher-gate.ps1` | Окно подтверждения при входе в Windows |
| `install.ps1` | Установка автозапуска + старт |
| `uninstall.ps1` | Остановка + снятие автозапуска |
| `stop-watcher.ps1` | Мягкая остановка |
| `status.ps1` | Состояние и лог |
| `show-windows.ps1` | Помощник для поиска hwnd окон |

Служебные файлы `watcher.log`, `watcher.pid`, `STOP` создаются рядом со скриптами во время работы.

## Безопасность

- Подтверждается **только** `Approve once` (клавиша `1`), «Approve always» не выбирается никогда.
- Диалог определяется по хвосту буфера — срабатывание на упоминание диалога в истории исключено.
- Скрипты открыты, без сетевых запросов, без внешних зависимостей — всё на встроенном PowerShell.
