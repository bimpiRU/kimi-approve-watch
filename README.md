> Автоапрув диалогов **«Approve once»** в [Kimi CLI](https://github.com/MoonshotAI/kimi-cli) + **стабилизатор ПК** на время долгой работы агентов в терминалах. Накатил одной командой — и машина под контролем, пока нейросети кодят и собирают проекты.

**EN:** Auto-approves `Approve once` permission dialogs of AI CLI agents in Windows Terminal, plus a PC stabilizer: keep-awake, high-performance power plan, RAM/disk/network monitoring — installed with a one-liner, confirmed by you at logon.

---

## Установка одной командой

```powershell
powershell -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/bimpiRU/kimi-approve-watch/main/quickstart.ps1 | iex"
```

Скачает репо в `%USERPROFILE%\kimi-approve-watch` (через `git clone`, а если git нет — ZIP) и запустит установщик. Повторный запуск команды обновляет до последней версии (`git pull`).

Необязательные переменные перед командой:

```powershell
$env:KAW_MODE = 'startup'   # gate | startup | none — пропустить меню выбора режима
$env:KAW_DIR  = 'D:\tools\kaw'  # куда ставить
```

## Что делает

### Наблюдатель апрувов

Kimi CLI при опасных действиях показывает интерактивный диалог (`Run this command? 1. Approve once ...`). Если агентов несколько — подтверждать приходится постоянно. Наблюдатель сканирует все окна Windows Terminal каждые 10 секунд и жмёт `1` за вас. Выбирается только **Approve once** — никаких «approve always».

- точное срабатывание: смотрит только хвост буфера терминала, свёрнутые окна не трогает
- чистит мусорный символ `1`, который TUI иногда оставляет в строке ввода
- профили агентов: `kimi` (по умолчанию), `claude` (экспериментально)

### Стабилизатор ПК

Держит машину в форме, пока агенты часами собирают APK и гоняют тесты:

- **keep-awake** — ПК не заснёт и экран не погаснет
- **High performance** — на время работы включает максимальную схему питания (троттлинг убивает долгие сборки), при выходе возвращает прежнюю
- **приоритет терминала** — WindowsTerminal держится на AboveNormal, не лагает под нагрузкой
- **RAM** — при нехватке пишет в лог топ-5 процессов по памяти
- **диск** — тревога, если места меньше порога (сборки едят гигабайты)
- **сеть** — фиксирует окна отказа: в это время API нейросетей недоступен
- **ожидающая перезагрузка Windows** — предупредит заранее, а не посреди билда
- **падение Windows Terminal** — алерт в лог, если сессии агентов могли погибнуть

Стабилизатор **ничего не ломает**: он только наблюдает и пишет лог. Единственные активные действия — схема питания (возвращается при выходе) и приоритет терминала (per-process, сбрасывается при перезапуске процесса).

### Надёжность самого инструмента

- **шлюз при входе в Windows** — работа начинается только после вашего «Да»
- **антидубль** — мьютексы не дают запустить два экземпляра
- **самовосстановление** — лаунчер перезапускает упавший модуль
- стоп одним файлом `STOP`, удаление одним скриптом

## Требования

- Windows 10/11
- Windows Terminal (агенты должны работать именно в нём)
- PowerShell 5.1 (встроен в Windows, ничего ставить не нужно)

## Режимы автозапуска

Установщик (`install.ps1`) предложит выбор:

| Режим | Что делает | Права админа |
|---|---|---|
| `gate` | При входе в Windows показывает окно подтверждения — старт только после «Да» | Да (UAC при установке) |
| `startup` | Ярлык в «Автозагрузке», стартует сразу и молча | Нет |
| `none` | Без автозапуска, просто запустить сейчас | Нет |

Там же включается/выключается стабилизатор (файл `stabilizer.enabled`).

## Управление

```powershell
.\status.ps1         # живы ли модули + хвосты обоих логов
.\stop-watcher.ps1   # остановить всё (мягко, через файл STOP)
.\uninstall.ps1      # остановить и снять автозапуск
.\show-windows.ps1   # список окон терминала: hwnd, заголовок, наличие диалога
```

## Настройка

### Наблюдатель

```powershell
.\watch-approve.ps1 -IntervalSeconds 5            # сканировать каждые 5 сек
.\watch-approve.ps1 -ExcludeHwnd 3344318          # не трогать окно с этим hwnd (своё!)
.\watch-approve.ps1 -Agents 'kimi,claude'         # профили агентов (claude — экспериментально)
.\watch-approve.ps1 -NoKeepAwake                  # не блокировать сон
.\watch-approve.ps1 -Once                         # один цикл и выход (для теста)
```

hwnd своего окна подскажет `show-windows.ps1` — исключите его, если не хотите автоапрува в личной сессии.

### Стабилизатор

```powershell
.\stabilize.ps1 -IntervalSeconds 15               # проверять ресурсы каждые 15 сек
.\stabilize.ps1 -MinFreeRamGB 2 -MinFreeDiskGB 10 # свои пороги тревог
.\stabilize.ps1 -WatchDrives 'C:','D:'            # контролируемые диски
.\stabilize.ps1 -HighPerformance -BoostTerminalPriority
.\stabilize.ps1 -Once                             # один цикл и выход (для теста)
```

При автозапуске стабилизатор стартует через `start-all.ps1` — флаги по умолчанию (`-HighPerformance -BoostTerminalPriority`) меняются в одной строке в начале этого файла.

## Как это работает

1. Раз в `IntervalSeconds` перечисляются окна класса `CASCADIA_HOSTING_WINDOW_CLASS` (Windows Terminal).
2. Через UI Automation читается текстовый буфер каждого окна (`TextPattern`).
3. Если в последних 15 строках есть все строки диалога из профиля агента — диалог активен.
4. Окно выводится на передний план (`AttachThreadInput` + `SetForegroundWindow`), отправляется `1` (`SendKeys`).
5. Если после апрува в строке ввода остался мусорный символ `1` — отправляется Backspace.
6. Параллельно стабилизатор опрашивает WMI/CIM (RAM, диски, CPU), пингует сеть, проверяет реестр на pending reboot и пишет в лог только **переходы состояний** — лог остаётся чистым.

## Почему не служба Windows?

Службы работают в сессии 0 и **не видят окна пользователя** — UI Automation и SendKeys там бесполезны. Поэтому всё запускается в интерактивной сессии: через задачу планировщика с триггером «при входе в систему» (режим `gate`, с окном подтверждения) или через «Автозагрузку» (режим `startup`). Это и безопаснее: без вашего входа и подтверждения автоапрув не работает.

## Безопасность

- Подтверждается **только** одноразовый апрув (клавиша `1`), «approve always» не выбирается никогда.
- Диалог определяется по хвосту буфера — срабатывание на упоминание диалога в истории исключено.
- Стабилизатор по умолчанию только наблюдает; активные действия — схема питания (восстанавливается при выходе) и приоритет процессов терминала (сбрасывается при их перезапуске).
- Профиль `claude` экспериментальный: включайте `-Agents 'kimi,claude'` осознанно и проверьте на своей версии Claude Code.
- Скрипты открыты, без сетевых запросов (кроме quickstart), без внешних зависимостей — всё на встроенном PowerShell.
